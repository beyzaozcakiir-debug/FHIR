# Contributing to FHIR

We welcome contributions to the FHIR project! Please follow the guidelines below to ensure a smooth contribution process.

## Contribution Guidelines
- Fork the repository and make your changes in a separate branch.
- Write clear and concise commit messages.

## Coding Standards
- Follow the existing code style in the project.
- Ensure that your code is well-documented and clean.

## Commit Message Format
- Use the format: `type: subject`.
- Types include: `feat` for new features, `fix` for bug fixes, and `docs` for documentation changes.

## Pull Request Process
1. Ensure your branch is up to date with the base branch before opening a pull request.
2. Open a pull request and link any relevant issues.

## Code Review Requirements
- All contributions must be reviewed by at least one other person before merging.
- Be open to feedback and ready to make adjustments as necessary.

Thank you for contributing!
